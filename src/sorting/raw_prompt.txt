Source exercises:
- Move all 0-tuples to beginning, 1-tuples to end, in O(n), no second list:
  Input: [(0,"x"), (1,12), (0,34), (1,90), (1,89), (0,"s"), (1,"7")]
- Counting sort: sort a billion ages (0-200) using an array of counters.
- Code Bubble Sort (animation: https://yongdanielliang.github.io/animation/web/BubbleSortNew.html),
  Insertion Sort, Quick Sort, Merge Sort. Calculate their complexity.
- n=1,000,000 → n^2 vs n·log(n) comparison discussion.

Pedagogical arc:
- Part 0: is_sorted, swap (in-place primitives)
- Part 1: bubble_pass → bubble_sort → count comparisons → discover O(n²)
- Part 2: insert_card → insertion_sort → best case O(n) on sorted data
- Part 3: merge (two sorted lists) → recursive merge_sort → n·log(n) via level argument
- Part 4: partition_01 (the 0/1 tuple exercise) → partition around pivot → quick_sort
- Part 5: count_values → counting_sort (billion ages, zero comparisons, O(n+k))
- Part 6: timing race with log-log plot; predict big-n runtimes; Timsort = merge + insertion
