Trees & Heaps chapter — from python_iitr_apr_24 (Jul 13.ipynb, Jul 19.ipynb, exercises.md).

Source exercises:
- Motivation: frequent add/delete/find. Why not list (find O(n))? Why not sorted list
  (insert O(n))? Why not dictionary (memory, no ordering)?
- Expression tree: 2*3+4 as Node(typ, val, left, right); eval recursively; parse trees,
  compilers (yacc/antlr pointers).
- Binary Search Tree: insert, find. Animation: https://www.cs.usfca.edu/~galles/visualization/BST.html
- Implement Deletion in BST (exercises.md).
- Imbalance on sorted input → AVL / Red-Black pointer:
  https://www.cs.usfca.edu/~galles/visualization/RedBlack.html; B-Trees for disk.
- Scheduler problem (cron): find task with earliest time. Six strategies compared
  (sort on insert, linked list, BST, giant per-second array, ...) → Min Heap.
- heapq: heapify, heappush, heappop.

Pedagogical arc:
- Part 1: expression trees — provided Node + print_tree; evaluate() recursively; build (2+3)*(10-4)
- Part 2: BST — insert (recursive, returns root), contains, in_order (discover tree sort!),
  find_min, delete (3 cases, successor trick), balance catch (sorted input → chain)
- Part 3: scheduler → min-heap as plain list: parent/left/right index arithmetic,
  heap_push (bubble up), heap_pop (sink down w/ smaller child), run_scheduler on
  (time, name) tuples, heap_sort, agreement with heapq
