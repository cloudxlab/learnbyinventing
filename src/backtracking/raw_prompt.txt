Source exercises:
- Q: There is a list of people, print the uniques hand shakes (["A","B","C"] -> A B, A C, B C)
- Q: Write a function to print all possible numbers of length n where each digit could go
  from 0 to m-1. Don't use itertools. generate(2,2) -> 00 01 10 11
  (both the iterative odometer next_numb(arr, m) version from Mar 9 and recursion)
- Q: Print all possible permutations of your name. Same length. perms("abc") -> 6 strings
- Q: Write a function to check if parenthesis are valid. are_valid("[]") True, "}{"  False
- Sudoku (Apr 4): board as 9x9 lists with 0 = empty.
  Implement find_first_empty_space(board) and can_place(i, row, col, board), then
  solve_sudoku(board) via backtracking: find empty; none -> True; try 1..9, can_place ->
  place, recurse, undo on failure; return False when all digits fail.

Pedagogical arc:
- Part 1: handshakes — nested-loop enumeration without repeats (n(n-1)/2)
- Part 2: odometer next_numb -> generate -> generate_rec (enumeration = counting,
  ties back to Learning to Count: B^d)
- Part 3: perms — choose first element, recurse on the rest (n!)
- Part 4: are_valid parentheses — stack as a fast validity checker
- Part 5: Sudoku — find_first_empty_space, can_place (row/col/3x3 block),
  solve_sudoku (try -> recurse -> undo). Provided: classic puzzle, print_board,
  is_solved checker. Closing: pruning vs 9^51; same skeleton as N-Queens and the
  glob `*` matcher from Pattern Matching.
